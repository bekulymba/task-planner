# Task Planner - Полный Список Проверок (Testing Checklist)

## 🎯 Edge Cases & Input Validation

### ✅ 1. Empty List (Пустой список)
**Как проверить:**
- [ ] Открыть приложение в новом браузере (очистить localStorage)
- [ ] Проверить: должно появиться сообщение "No tasks yet. Click 'New Task' to create one!"
- [ ] Проверить: фильтры и сортировка не вызывают ошибок
- [ ] Открыть DevTools Console - не должно быть ошибок

**Ожидаемый результат:** Приложение показывает дружелюбное сообщение, работает без ошибок

---

### ✅ 2. All Tasks Completed (Все задачи завершены)
**Как проверить:**
- [ ] Создать 3-5 задач
- [ ] Отметить все задачи как "Completed"
- [ ] Выбрать фильтр "Active" - должно появиться "No tasks match the current filter"
- [ ] Выбрать фильтр "Completed" - все задачи видны
- [ ] Попробовать все комбинации фильтров (Failed, All)

**Ожидаемый результат:** Фильтры работают корректно, пустые фильтры показывают сообщение

---

### ✅ 3. Invalid User Input (Неверный ввод)

#### Test 3.1: Пустой заголовок
**Как проверить:**
- [ ] Нажать "New Task"
- [ ] Оставить поле "Title" пустым
- [ ] Нажать "Create"
- [ ] Должно появиться уведомление: "Title is required"

#### Test 3.2: Слишком длинный заголовок
**Как проверить:**
- [ ] Нажать "New Task"
- [ ] Ввести заголовок длиной >50 символов (например, скопировать 60 раз букву "a")
- [ ] Нажать "Create"
- [ ] Должно появиться: "Title is too long (max 50 characters)"

#### Test 3.3: Слишком длинное описание
**Как проверить:**
- [ ] Нажать "New Task"
- [ ] Ввести корректный заголовок
- [ ] Ввести описание длиной >300 символов
- [ ] Нажать "Create"
- [ ] Должно появиться: "Description is too long (max 300 characters)"

#### Test 3.4: Только пробелы в заголовке
**Как проверить:**
- [ ] Нажать "New Task"
- [ ] Ввести только пробелы в поле "Title": "     "
- [ ] Нажать "Create"
- [ ] Должно появиться: "Title is required"

**Ожидаемый результат:** Все некорректные данные отклоняются с понятными сообщениями

## 🚨 Error Handling

### ✅ 7. Invalid URL Parameters (Неверные параметры URL)

#### Test 7.1: Неверный фильтр
**Как проверить:**
- [ ] Открыть URL: `http://localhost:3000/?filter=invalid&sort=date-asc`
- [ ] Приложение должно работать
- [ ] URL должен обновиться на корректный: `?filter=all&sort=date-asc`
- [ ] Консоль должна показать warning: "Invalid filter parameter: invalid"

#### Test 7.2: Неверная сортировка
**Как проверить:**
- [ ] Открыть URL: `http://localhost:3000/?filter=all&sort=wrong`
- [ ] Приложение должно работать
- [ ] URL должен обновиться на корректный
- [ ] Консоль показывает warning

#### Test 7.3: XSS атака через URL
**Как проверить:**
- [ ] Открыть URL: `http://localhost:3000/?filter=<script>alert('xss')</script>`
- [ ] Скрипт НЕ должен выполниться
- [ ] Приложение работает нормально

**Ожидаемый результат:** Неверные параметры игнорируются, приложение работает

### ✅ 8. Browser Storage Errors (Ошибки хранилища)

#### Test 8.1: localStorage заполнен
**Как проверить:**
- [ ] Открыть DevTools → Console
- [ ] Выполнить код для заполнения localStorage:
  ```javascript
  try {
    for(let i = 0; i < 10000; i++) {
      localStorage.setItem('test' + i, 'x'.repeat(100000));
    }
  } catch(e) {
    console.log('Storage full:', e);
  }
  ```
- [ ] Попробовать создать новую задачу
- [ ] Должно появиться: "Failed to save task"
- [ ] Задача НЕ должна быть создана

#### Test 8.2: localStorage отключен
**Как проверить:**
- [ ] В браузере отключить cookies и site data (Settings → Privacy)
- [ ] Перезагрузить страницу
- [ ] Попробовать создать задачу
- [ ] Должна появиться ошибка: "Failed to save tasks"
- [ ] Консоль показывает: "Error saving tasks"

#### Test 8.3: Поврежденные данные в localStorage
**Как проверить:**
- [ ] Открыть DevTools → Application → Local Storage
- [ ] Найти ключ `tasks`
- [ ] Изменить значение на: `{broken json`
- [ ] Перезагрузить страницу
- [ ] Приложение должно загрузиться с пустым списком
- [ ] Консоль показывает: "Error loading tasks"

**Ожидаемый результат:** Ошибки обрабатываются, пользователь видит сообщения
